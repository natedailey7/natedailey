Game Projects

Game Projects

During spring 2022, I was in a game engine programming class at Pomona College. Most game design takes place in pre-built engines, which deal with low-level details such as basic physics, lighting, collisions, etc. (such as the Unity or Unreal engines). However, in this class we were assigned to create a 2d-graphics engine and game, and a 3d game based on the bare-bones Frenderer engine (created by Professor Joseph Osborn). Both games were created in teams of 3, and we had a lot of fun putting them together. Following are samples of both games: Triumvirate Arena and JumpyBall!

Triumvirate Arena

Triumvirate Arena is a battle card game featuring three players: Nate, Chloe, and Grace. Each player type has three signature moves which might increase or decrease your health/mana, and/or do damage to the other player. The goal of this two player game is to reduce the opponent to zero health.

We wrote Triumvirate Arena in Rust. And we did it from scratch, without a pre-made game engine. In our team of 3 (myself, Chloe, and Grace), I worked on gameflow mechanics (health/mana interactions, turn taking), player moves, and creating original music, while my partners worked on other various mechanics and graphics.

Click here for the Triumvirate Arena repository, in order to see the code and more gameplay examples!


Features:

  • Two player turn taking
  • Choose player
  • Choose move
  • Animated health bars
  • Fonts + text writing
  • Bitblt images
  • Original music
  • Sound effects for selecting moves, players, etc.
  • Main/end screen
  • Get info screens for players and moves
  • Health and mana tracking (generation and subtraction moves)
  • Health and mana requirements for moves

JumpyBall

JumpyBall is a 3D parkour game. The player navigates through levels by jumping across platforms, avoiding the ground, and working their way to the end gem.

While Triumvirate Arena was created from scratch, JumpyBall was created with the help of the Frender game engine (created by Professor Joseph Osborn), which assisted with the 3D rendering. We were left to create the physics, collision system, assets, and more.

In JumpyBall, I primarily worked on asset creation and the collision system. I created all of the 3D models in Blender and also wrote a Python script in Blender to determine the bounding boxes for each object (the core of our collision system).

Features:

  • Rolling
    • Mid air rolling
    • Roll while jumping
  • Jumping (Double jump!)
  • Collision
    • Bounding boxes for every object in scene, created via Blender Python script
    • AABB-Sphere collision
  • Camera
    • Mouse responsive camera
    • Mouse responsive out of window
  • Level Switching
    • Switch between levels (and beat the game) by collecting gems
    • Level restart when player falls
  • Modeling
    • Made-from-scratch 3d models of ball, landing objects, and world
  • Textures
    • Cosmetic map textures (ex. Lava ground, space sphere)
    • Ball texture (the Death Star!)
    • Landing object textures (pillars and cubes)
  • Music
    • Original JumpyBall music!
